🛡 SIEM - Splunk

🔗 [SIEM-Splunk](https://github.com/risha463/SIEM-Splunk)

📄 Incident Response Report — Task 2 (SIEM Analysis Using Splunk)

🔐 1. Overview

This project analyzes Windows Security Logs using Splunk SIEM to detect suspicious authentication attempts, identify brute-force behavior, classify incidents, and provide remediation steps.

📁 2. Log Source Details

  • Log File: windows_security_logs.txt
  • Log Type: Windows Security Authentication Logs
  • Sourcetype: log2metrics_keyvalue
  • Index Used: task2_index

🎯 3. Objectives

  • Ingest logs into Splunk
  • Monitor authentication events
  • Detect failed & successful logins
  • Identify brute-force login patterns
  • Build visualizations
  • Prepare SOC-style incident report

🔍 4. Splunk Queries Used

  • ✔ View All Events index="task2_index"
  • ✔ Failed Login Attempts index="task2_index" "Failed Login"
  • ✔ Successful Login Attempt index="task2_index" "Successful Login"
  • ✔ Count of Failed Logins index="task2_index" "Failed Login" | stats count
  • ✔ Failed Logins by IP Address index="task2_index" "Failed Login" | stats count by IpAddress
  • ✔ EventID Breakdown (4624 / 4625) index="task2_index" | stats count by EventID

📊 5. Visualizations Created

  • ✔ EventID Breakdown Pie Chart
  • ✔ Failed Login Attempts Bar Chart
  • ✔ Successful vs Failed Logins
  • ✔ Status Pie Chart
  • ✔ Account Name Pie Chart

🚨 6. Findings (Alert Analysis)

🔹 Summary of Events:
EventIDMeaningCount
4625Failed Login3
4624Successful Login1
🔹 Suspicious Indicators:
  • Same attacker IP: 185.34.55.1
  • Target account: admin
  • 3 failed attempts within seconds
  • Followed by 1 successful login
  • 👉 This is a confirmed brute-force attack pattern.

🛑 7. Incident Classification

  • Incident Type: Brute-Force Authentication Attack
  • MITRE ATT&CK: T1110
  • Severity: High
  • Reason: Rapid failed attempts + final successful compromise.

🛠️ 8. Recommended Actions

✔ Immediate:
  • Block IP 185.34.55.1
  • Force password reset of admin
  • Review session after compromise
  • Enable MFA
✔ Policy Fixes:
  • Enable Account Lockout Policies
  • Monitor EventID 4625 spikes
  • Restrict admin login from external IPs
  • Close Project

🎯 Phishing Website Detector

  • Phishing_website.ipynb → Main Python code (Colab Notebook)

📊 Dataset Information

The dataset includes features like:

  • UsingIP → Website has IP address instead of domain
  • LongURL → Is the URL suspiciously long?
  • PrefixSuffix → Uses hyphen in domain name?
  • SubDomains, HTTPS, DomainRegLen

Random Forest Classifier

Hands-On Experience

My strongest practical experience comes from cybersecurity projects, labs, and internship-based security testing. I explored real attack techniques, defensive monitoring, and automation through hands-on work rather than only theory.

Web Application Security Testing

Through hands-on labs and practical security testing, I explored how attackers exploit web application vulnerabilities and how defenders detect and respond to them. I studied SQL Injection techniques including authentication bypass, UNION-based injection, error-based injection, and time-based SQL Injection to understand how databases can be manipulated by malicious input. I also learned how to identify suspicious patterns, malicious payloads, unusual URLs, and attack indicators through logs and monitoring activities. In addition, I researched Cross-Site Scripting, Command Injection, IDOR, RFI, and LFI attacks to understand how attackers target web servers and user input fields. Using TryHackMe, LetsDefend, and DVWA, I practiced detecting attack behavior, analyzing web requests, and monitoring indicators of compromise in real environments.

SOC Analysis & Log Monitoring

Through practical SOC labs and monitoring simulations, I learned how security analysts investigate suspicious activity using Linux logs and SIEM platforms like Splunk. I practiced log monitoring, alert investigation, authentication analysis, and threat detection by reviewing system events, failed logins, unusual processes, and suspicious behavior patterns. Using Splunk, I worked with log searches, event correlation, and real-time monitoring to understand how analysts detect attacks and investigate alerts in security environments. I also improved my Linux investigation skills by analyzing running processes, active services, and suspicious system activity using commands such as ps aux and systemctl. Through LetsDefend and TryHackMe labs, I gained experience investigating brute-force attempts, suspicious logins, attacker activity, and possible compromises.

Wireshark Network Traffic Analysis

Through hands-on Wireshark labs and network analysis exercises, I learned how network traffic moves between systems and how security analysts inspect packets to detect suspicious activity. I practiced analyzing protocols such as DNS, HTTP, TCP, and ARP while understanding how data is transmitted across networks. I also explored packet flow behavior, dropped packets, communication patterns, and how attackers misuse network protocols during attacks. I researched attacks such as Man-in-the-Middle and ARP spoofing to understand how malicious traffic can be identified through abnormal MAC and IP behavior. Using packet inspection and traffic analysis, I learned how to investigate suspicious connections, detect insecure plaintext communication, and recognize indicators of compromise within network traffic.

Python Security Automation

Through Python-based security projects and scripting practice, I learned how automation can improve threat detection, log analysis, and security monitoring. I practiced writing Python scripts to extract IP addresses, analyze patterns, detect suspicious activity, and process logs from different operating systems. I also explored how Linux and Windows logs use different formats and how scripts can be adapted for security investigations and monitoring tasks. I developed practical projects such as phishing website detectors, threat detection scripts, and secure authentication systems to strengthen my understanding of cybersecurity automation. These experiences helped me improve my scripting, problem-solving, and security analysis skills through hands-on implementation and real-world scenarios.

Phishing Email Analysis

Through phishing analysis labs and practical investigations, I learned how attackers use spoofed emails, fake domains, malicious links, and social engineering techniques to target users. I practiced analyzing suspicious emails by inspecting headers, attachments, embedded links, sender information, and domain behavior to identify indicators of phishing activity. I also explored how attackers use hidden tracking images and malicious content to detect whether an email has been opened and whether a target is active. Using tools such as ANY.RUN, VirusTotal, WHOIS Lookup, and MX record analysis, I practiced investigating domains, verifying mail servers, analyzing suspicious files, and detecting malicious behavior. I also improved my understanding of SPF, DKIM, and DMARC validation, email header analysis, and phishing detection techniques through hands-on labs and real-world simulations.
